# Хроники Восьми Земель

Кликабельный фронтенд для проведения авторских D&D-кампаний на одном общем экране. Мастер открывает сервис, показывает игрокам карту и ведёт приключение без регистрации, комнат и отдельного мастерского интерфейса.

## Что уже работает

- мировая карта из восьми кликабельных регионов;
- отдельная страница героев с книжными разворотами;
- перелистывание легенды, характеристик и навыков персонажа;
- завершённая кампания Нор’Иль’Скальда с пересказом сюжета;
- страницы-заготовки для будущих кампаний;
- броски `d4`, `d6`, `d8`, `d10`, `d12` и `d20`;
- адаптивная вёрстка без авторизации и серверной части.

## Технологии

- React 19;
- TypeScript;
- Vite;
- CSS Modules;
- FSD-архитектура;
- npm и `package-lock.json`.

## Локальный запуск

Требуется актуальная LTS-версия Node.js.

```bash
npm ci
npm run dev
```

После запуска Vite выведет локальный адрес приложения.

### Команды

```bash
npm run dev          # локальная разработка
npm run typecheck    # проверка TypeScript
npm run build        # production-сборка в dist/
npm run preview      # просмотр локальной production-сборки
npm run build:pages  # сборка для GitHub Pages
```

## Навигация приложения

- `/` — карта Восьми Земель;
- `/heroes` — книга героев;
- `/region/<region-id>` — летопись выбранного региона.
- `/roadmap` — продуктовый план развития проекта.

Маршруты управляются React Router. Для прямого открытия внутренних страниц GitHub Pages публикует SPA fallback, а после переезда в Yandex Cloud fallback на `index.html` настраивается в API Gateway.

## Архитектура фронтенда

```text
src/
  app/         инициализация приложения
  pages/       карта, герои и страница региона
  widgets/     крупные интерфейсные блоки
  features/    навигация и игровые действия
  entities/    модели и UI героев, кампаний и регионов
  shared/      конфигурация, утилиты, ассеты и глобальные стили
```

Правила проекта:

- один компонент — один файл;
- стили компонентов хранятся в `*.module.css`;
- цвета задаются только через переменные из `src/shared/styles/variables.css`;
- функции и преобразования данных выносятся в `shared/lib` или модель соответствующего слоя;
- интерфейс читает канонические данные из `content/`.

## Контент

```text
content/
  characters.json             герои
  races.json                  расы
  rules.json                  домашняя облегчённая d20-система
  world-map.json              регионы и интерактивные полигоны карты
  campaigns/
    nor-il-skald.json         данные кампании
    nor-il-skald-guide.md     сценарий для мастера
```

Первая завершённая летопись — «Бубсильда и проклятие Нор’Иль’Скальда».

## Визуальные материалы

`assets/concepts/` содержит утверждённые портреты, карту мира, изображения регионов, референсы рас и стиль книжных листов. Индекс ассетов находится в `assets/concepts/manifest.json`.

## Локальные Codex-скиллы

- `$maintain-dnd-world` — целостность схемы и связей;
- `$create-dnd-character` — создание героя;
- `$create-dnd-enemy` — создание врага или босса;
- `$create-dnd-campaign` — создание кампании для региона.

Правила работы агентов описаны в `AGENTS.md`.

## Публикация на GitHub Pages

Репозиторий подготовлен к автоматическому деплою через `.github/workflows/deploy-pages.yml`.

1. Отправьте актуальную ветку `main` на GitHub:

   ```bash
   git push -u origin main
   ```

2. Откройте репозиторий `etozhenerk/DnD` на GitHub.
3. Перейдите в **Settings → Pages**.
4. В разделе **Build and deployment → Source** выберите **GitHub Actions**.
5. Откройте вкладку **Actions** и дождитесь завершения workflow `Deploy to GitHub Pages`.

После публикации приложение будет доступно по адресу:

```text
https://etozhenerk.github.io/DnD/
```

Workflow запускается при каждом push в `main`. Для ручного повторного запуска доступен `workflow_dispatch` во вкладке Actions.

Если репозиторий будет переименован, необходимо изменить `base` для режима `github-pages` в `vite.config.ts`.

## Ограничения MVP

- нет авторизации и игровых комнат;
- состояние бросков и открытых страниц не сохраняется;
- мастер управляет страницей и показывает её игрокам через общий экран;
- полноценная кампания пока создана только для Нор’Иль’Скальда.
