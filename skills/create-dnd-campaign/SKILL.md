# Создание кампании D&D

Создавать самостоятельную кампанию для кликабельной области карты, совместимую с общим миром и текущими данными фронтенда.

## Контекст

Прочитать `AGENTS.md`, весь базовый `content/`, `assets/concepts/manifest.json`, список героев и существующие кампании. Осмотреть соответствующую область по пути актуальной карты из `content/world-map.json`. Не копировать сюжет Нор’Иль’Скальда под новыми именами.

## Воронка вопросов

Задавать не более трёх вопросов за раз. Сначала искать ответ в каноне.

Обязательно выяснить:

1. Какая область карты открывается: её тема, визуальный образ и рабочее название?
2. Какая длительность нужна: короткая сцена, one-shot на 3–6 часов или несколько встреч?
3. Какие герои играют и есть ли обязательный личный мем, конфликт, злодей или финальный образ?

Затем при необходимости уточнить:

- линейная структура, развилка или свободный выбор локаций;
- желаемая доля боя, исследования, загадок и диалогов;
- тон и запрещённые темы;
- желаемый тип финала: победа, выбор, трагикомедия, клиффхэнгер;
- нужны ли новые расы, герои, изображения или только контент кампании;
- какая информация видна игрокам до открытия области.

Если пользователь делегирует выбор, создавать one-shot на 4–6 часов для пяти текущих героев: 5–7 локаций, 2 социальных/исследовательских испытания, 2–3 боя, один лейтенант и двухфазный босс.

## Каркас кампании

1. **Обещание области:** одна фраза о фантазии, которую получат игроки.
2. **Конфликт:** злодей хочет изменить мир; без героев произойдёт конкретная катастрофа.
3. **Крючок:** личная причина минимум для двух героев.
4. **Маршрут:** локации с видимым прогрессом и хотя бы одной осмысленной развилкой.
5. **Три столпа:** бой, исследование и социальное решение представлены в разных пропорциях.
6. **Повторяющийся мотив:** визуальный или комедийный элемент связывает сцены.
7. **Финал:** ранние решения меняют условия босса или итог.
8. **Эпилоги:** персональный результат для каждого участвующего героя.

## Варианты структуры

- **Короткая область:** 1–2 локации, одна проблема, 30–90 минут.
- **One-shot:** 5–7 локаций, 6–10 встреч, 3–6 часов.
- **Мини-кампания:** 3 акта, 10–18 локаций, сохранение состояния между встречами.
- **Узел карты:** одна центральная локация и 3–5 направлений, каждое меняет финал.

## Производство файлов

1. Согласовать краткий концепт, если пользователь не поручил полный автономный результат.
2. Создать `content/campaigns/<id>.json` по существующей форме.
3. При необходимости применить `$create-dnd-enemy` и `$create-dnd-character`.
4. Создать синхронный `content/campaigns/<id>-guide.md` с текстом для мастера.
5. Добавить только те новые мировые факты, которые нужны кампании.
6. Применить `$maintain-dnd-world` для финальной проверки.

## Критерии готовности

- Каждая локация имеет цель, атмосферу, действие и переход.
- Провал создаёт осложнение, но не блокирует продолжение.
- У ключевой встречи есть минимум два подхода.
- Финальный босс связан с механиками и решениями предыдущих сцен.
- JSON и гайд согласованы; ссылки и `id` валидны.
- Пользователю показаны краткий синопсис, маршрут, новые сущности, вариативность и открытые вопросы.
